Details

Nutty uses WebRTC to share the terminal with peers in the browser. WebRTC is true peer-to-peer technology built into browsers to enable realtime communication between peers. All data transmitted using WebRTC is encrypted. Nutty uses WebRTC's DataChannel to share terminal peer-to-peer. Nutty uses PeerJS javascript library which provides abstraction over WebRTC APIs.

Nutty's web based terminal emulator (hterm) uses Chrome's Native Messaging API to communicate with the nutty.py script which creates pseudo terminals.

Browser supported: Chrome 29+. Firefox support coming soon!

Security notes

  • Communication between peers is direct p2p without any proxy and encrypted by WebRTC layer
  • Access to backend script nutty.py is restricted to nutty's chrome extension. Access to nutty's chrome extension is in turn restricted to https://nutty.io domain
  • Any terminal recorded and uploaded will not contain passwords as they are not echo'ed to the terminal

FAQ

  • Q: Why nutty when we already have tools like tmux and screen?

    tmux and screen works when the other person has ssh access to your machine. But if you or your peer is behind the firewall or if you don't want to share username/password credentils when you just want to share terminals - nutty is the answer.

  • Q: Installation of nutty on Chrome requires sudo root access. why?

    I have raised this issue with Chromium team: https://code.google.com/p/chromium/issues/detail?id=308854

  • Q: Will closing my browser tab used for sharing terminal automatically disconnect peer?

    Yes.

  • Q: Will nutty be supported on Firefox?

    Yes. Soon.

  • Q: Will nutty be supported on Windows?

    If there are enough requests ssh client can be made to work on browser which in turn can be sharable. So a Sharer will ssh-login to a server and share the link

  • Q: Can I share only local machine's terminal? How can I share my access to a server in the cloud?

    In nutty's terminal you can just ssh to the remote server. The same shared link will work(duh).

  • Q: When I record terminal using nutty are my passwords captured too?

    Since passwords are not echoed back to the terminal it never gets recorded.

  • Q: Why does nutty use WebRTC?

    Nutty is one of the examples of the cool things that are possible with WebRTC DataChannels. WebRTC DataChannels are encrypted and p2p and hence completely secure.
